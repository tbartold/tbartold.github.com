Working with Polish/Central European Accents in Windows

Task 1: Adding Multi-lingual capabilities (5 Steps):

The first step in using Windows 98 (or Windows 95) with international accented characters is making sure you have installed/enabled Multi-Lingual capabilities. There were lots of solutions for Windows 3.1 out there, but none of them work well with the newer operating systems. Some of the third party software available is probably worth looking at, but I have found the "native" fonts and keyboards to work best in the majority of situations.

Step 1: adding multi-lingual fonts

  • click on "start" - "settings" - "control panel".
  • then double-click on "add/remove programs".
  • click the "windows setup" tab.
  • scroll down to "multi-language support".
  • double click it to get the language selection menu.
  • if there is no check mark, click on "central european" to add it.
  • click OK.
  • click OK again, and it should add the language support.
Step 2: adding the polish keyboard mapping (similar for other "CE" languages)
  • again "start" - "settings" - "control panel".
  • double-click on "keyboard".
  • select the "language" tab. (make sure there is a check by "enable indicator on taskbar" the last line in that window)
  • click the "add" button (middle of the window).
  • a little language pop-up appears... scroll down to the language you want to add: e.g. "Polish".
  • hit Ok to close the pop-up.
  • There should now be English and Polish in the Language box.
  • Highlight "Polish".
  • then click the "properties" button to select a different layout. (another pop-up).
  • select "Polish (programmers)" (for how to use it, see Step 5 below).
  • Click the OK button.
  • (Note which key-combo is selected for switching between languages: [alt]+[shift] by default)
  • Click OK again.
Step 3: switching keyboards
  • Now, at the bottom right of your screen (next to the clock) you should now have a "En" indicator, indicating that you are using the english keyboard.
  • Press [alt]+[shift] together to make it change to "Pl"
  • Press [alt]+[shift] together again to make it change back to "En"
  • Step 4: selecting fonts
  • You will be able to use the Central European versions of Times, Courier and Arial without further ado. If you want more fonts go to the microsoft fonts page and select other multi-lingual fonts to download. http://www.microsoft.com/typography/fontpack/default.htm
Step 5: check it out
  • Press [alt]+[shift] together again switch to the "Pl" Keyboard mapping.
  • Go to Word (if you have it) and type in [ctrl]+[alt]+c (or type ~c) and you should get a lower case "c" with accent. If you get a cirlce-c, you are using the english keyboard, or a font that is not multi-lingual. Not all fonts are multi-lingual, but there are a lot to choose from. (see step 4 above)
  • How to type the letters: In order to type polish accented letters with a physical english keyboard, the "Polish (Programmers)" keyboard mapping enables two methods (both commonly used in Poland).
  • By Key Combo: simultaneously type [ctrl]+[alt]+(letter) where (letter) is one of: a, c, e, l, o, n, s, z, or x for the appropriate accented letter. [ctrl]+[alt]+[shift]+(letter) gives the upper case accented letter. Note: There are some programs or computers where not every key combo will work. e.g. on HP Pavilion systems, the key combo "[ctrl]+[alt]+s" will produce a "support info box". I talked to HP about this, and they say it is coded into the hardware, and that's just the way it is. In Family Tree Maker, a [ctrl]+[alt]+c will compress the database, and I don't know how to turn that off.
  • Using "tilde": Another common way to type accented letters is by typing the tilde (~) key, followed by a letter that you want accented. This is not normally activated on english keyboards, but with the Polish keyboard mapping (this one included), typing a single tilde key will not produce any visible character. If one follows the tilde with any of the accentible letters, it will produce the properly accented letter. If you type tilde followed by any other letter, it will produce the tilde followed by that letter (or number etc.). If you type tilde followed by space, it will just produce the tilde. Note: Using tilde works for every letter with every program and computer I've tried this with, so this is probably the better method to use.
  • Note: There are two accented z's in Polish, with a dot above or with an accent mark above. The z-dot is created by using the z key, the less common z-accent is created with the x key.

Task 2: Getting old style programs to acknowledge Central European Fonts.

The main problem is that although one can type polish letters very easily under Windows 95 or Windows 98, FTM and other "older" programs refuse to display them as polish accented letters. Word for Windows, WordPerfect, Netscape, Explorer, etc. etc. work fine with displaying the letters.

Before you start, you will need to activate/install multi-lingual capabilities in Windows. In case you don't know how, see task 1 above.

Now you will need to modify your WIN.INI file.

The first step in displaying the Central European letters is making FTM and other Windows based programs think that there is a special Central European font on the computer, when there really isn't. The Central European accented letters are contained in the standard Windows multi-lingual fonts. Unfortunately, older programs just don't know how to find them. So here's how to make them find them. I've never received a good explanation of why this works, but it does.

This info came from a company called "Young Digital Poland" who market a super Polish <-> English Dictionary in conjunction with Harper Collins.

Adding the following entries to the C:\WINDOWS\WIN.INI file are necessary for the ability to display Central European accented letters:

[FontSubstitutes]
Arial CE,238=Arial,238
Courier New CE,238=Courier New,238
Times New Roman CE,238=Times New Roman,238
Arial CE,0=Arial,238
Courier New CE,0=Courier New,238
Times New Roman CE,0=Times New Roman,238
At this point you should restart windows, so that it will know what "Arial CE" refers to.

Task 3: Getting Family Tree Maker to Display Polish Accented Letters

Since I have never gotten an answer to my questions about being able to DISPLAY Central European Accented letters in FTM 5, I have struggled with it on my own. Adding the "Central European" fonts to Windows was the first big step (task 2 above).
"Family Tree Maker is not designed or tested to run in modes other than in the English version of Windows and keyboards. If the character is not available in Character Map with the standard fonts, then it can't be entered into Family Tree Maker." - Paul Burchfield, The Learning Company (21 Feb 1999).
I haven't got a perfect solution, but here's an almost complete solution.

Getting the fonts to work on the Family Page is is the most important part. Having them display properly on the reports is fine, but you need to display them on the "FamPg" so you know that what you've typed in is correct!

FTM does not provide an easy way to change the display font for the FamPg or the Index. To change the font on the FamPg, you need to edit the "FTW.INI" file, which is located in the FTM directory (usually "C:\Program Files\Family Tree Maker\").

There should be two lines (among others) in the "Options" section of the file that specify the fonts to use on the "FamPg".

[Options]
...
FamPagFont=Arial
TabsFont=Arial
...
Change these two lines, and these two only, to use the Central European fonts. i.e.: 
[Options]
...
FamPagFont=Arial CE
TabsFont=Arial CE
...
Now you can start up Family Tree Maker, and the "FamPg" will show the proper accented letters as you type them in (make sure you select your polish keyboard first).

To select the "Central European" fonts on the reports is the really tedious part.

One needs to select each of the buttons in turn: Journal; Ancestor (Fan and Standard separately), Descend (Fan and Standard), Hrglass (Fan and Standard), GenRpt (see note 2 below), FamGrp, and Report.

For Each Report, you will need to select "Format" and then "Text Font, Style and Size..." from the menu. That will open up the font selection window. You will need to select each and every item in the "Items to Format" box one by one, and then select the proper font (i.e.. "Times New Roman CE (TT)" or "Arial CE (TT)") in the "Font" box.

Note 1: There is no possibility of setting the Display Font for the Index selection.

Note 2: Changing the fonts in the "GenRpt" section seems to have no effect. i.e. the letters are not displayed correctly.

That's the best I've been able to do so far. I still have no idea how to get the Index to list the central european names correctly, and I have no idea why changing the Fonts in the GenRpt has no effect.

I would still like to get an answer from the makers of the program, but maybe someone else out there can go further with this?
